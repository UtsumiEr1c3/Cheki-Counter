## Why

地下偶像切奇(拍立得)需要按偶像统计数量和金额,当前用 Excel/CSV 手工记录麻烦、无法看饼图/折线图、也无法在 livehouse 现场快速加一条。需要一个离线可用、支持 CSV 导入导出的 Android App,把"加一条切奇记录"和"看数据"这两件事做顺。

## What Changes

- 新建 Flutter Android App(输出 APK),本地 SQLite 存储,无后端。
- 主界面网格展示所有"有切奇记录"的偶像卡片,顶部显示总切数 / 总偶像数 / 总金额,支持按切数或按金额排序。
- 每张卡片边框 = 偶像应援色;卡片右上角 `+` 弹出添加切奇 popup(偶像名/应援色/团体 锁定,仅填日期、数量、单价、场地)。
- 右下角 `+` 新建偶像 popup:名字、应援色、团体可改;同时必须填第一条切奇记录(无记录的偶像不存在)。
- 点击偶像卡片 → 个人统计页:切数 + 总金额、切数折线图(按日 / 按月可切换)、按日期倒序的记录列表(可删单条)。
- 右上角统计页:应援色饼图 + 按金额 / 按切数排行,支持按自然年筛选(下拉仅列出有数据的年份)。
- 设置页:CSV 导入(合并到现有数据)、CSV 导出;入口另有"团体总览"显示每个团体的总切数。
- 数据模型业务主键 =(名字 + 应援色 + 团体)三元组;改名 / 改色 / 换团 = 新建偶像,不支持原地编辑偶像字段。
- 删除仅作用于单条切奇记录;偶像最后一条记录被删后自动消失。
- CSV 格式:`ID,应援色,团体,日期,数量,单价,小计,场地,创建时间`,与仓库根目录 `counts.csv` 保持一致。

## Capabilities

### New Capabilities

- `idols`: 偶像聚合视图 —— 基于切奇记录派生的偶像集合、主界面卡片、新建偶像(附带首条记录)、按三元组合并、最后一条记录删除时自动清理。
- `records`: 切奇记录的增删 —— 添加 popup、默认单价策略、单条记录删除。
- `statistics`: 全局与单人统计 —— 应援色饼图、金额/切数排行、年份筛选、个人切数折线图(按日/按月)、团体总览。
- `csv-io`: CSV 导入导出 —— 列格式、合并规则、导出时的行顺序与编码。
- `settings`: 设置页 —— 导入 / 导出 / 团体总览入口。

### Modified Capabilities

(无,首次提案,不涉及已有 spec)

## Impact

- **新仓库内容**:Flutter 项目骨架(`lib/`、`pubspec.yaml`、`android/` 签名配置),SQLite schema 两张表(`idols`、`records`)。
- **依赖(预估)**:`sqflite`、`path_provider`、`file_picker`(导入)、`share_plus` 或 SAF(导出)、`fl_chart`(饼图+折线图)、`csv`(解析)、`intl`(日期)。
- **数据兼容**:仓库根 `counts.csv` 作为导入基准数据,列顺序已对齐新 schema。
- **平台**:仅 Android(APK);iOS/桌面暂不构建,但代码不写 Android 专有 API,保留未来可扩展的空间。
- **无破坏性变更**:首个版本,没有历史用户数据需要迁移。
