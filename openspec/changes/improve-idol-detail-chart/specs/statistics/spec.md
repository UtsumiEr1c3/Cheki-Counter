## MODIFIED Requirements

### Requirement: 个人详情页 - 切数折线图

个人详情页 SHALL 包含一个切数折线图,提供"按日"和"按月"两种模式供用户切换。折线与下方填充区 SHALL 使用可读性友好的颜色——当偶像应援色为浅色（相对亮度 > 0.7）时,图表渲染前 MUST 将该色在 HSL 空间的 lightness 降至不超过 0.45,以保证在浅色卡片背景上清晰可见。

#### Scenario: 按日模式相邻记录点全连接

- **WHEN** 用户选择"按日"
- **THEN** X 轴只包含该偶像实际有记录的日期（不补零),Y 轴等于当日 `SUM(count)`,所有相邻记录点 MUST 以直线连接成一条连续折线

#### Scenario: 按月模式连续连线

- **WHEN** 用户选择"按月"
- **THEN** X 轴为从最早到最新月份的连续月序列,缺失月份 Y 值补 0,所有点以连线连接

#### Scenario: 浅色应援色折线自动降亮

- **WHEN** 偶像应援色的相对亮度 > 0.7（如白色、金色、黄色）
- **THEN** 折线色与 `belowBarData` 填充色 MUST 使用 HSL lightness ≤ 0.45 的降亮版本,而非原始应援色

#### Scenario: 深色应援色保持原色

- **WHEN** 偶像应援色的相对亮度 ≤ 0.7（如红色、蓝色、紫色）
- **THEN** 折线与填充色 MUST 使用原应援色,不做降亮处理
